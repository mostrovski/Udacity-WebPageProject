Web Basics

Web basically includes client computers, browsers, server computers, and the Internet. Browsers are special programs that help client computers connect (communicate) to server computers. Let's say somebody wants to open a website. He types the name of the website and presses the "Enter" button. The browser sends a request to the server computer, where this website is hosted. The server responds with the code of the web page that was requested (I guess), whereafter the browser processes the code, and shows the page to the client the way a human being is used to - without the code :) If I understand the things correctly, the same way one can get the access to various data, not necessarily web pages. There may be the direct access to different types of files that may be not embedded into web pages.

The Internet is the network that unites all the computers mentioned above. HTTP - HyperText Transfer Protocol - is the protocol that computers use to "communicate" (exchanging data on request) on the Web. HTML - HyperText Markup Language - is the most common data type on the Web. As a "language", it says browser how to show the web page to the client.

HTML Basics

One can tell a browser how to display the content with the help of so-called HTML elements. HTML element usually contains the opening and closing tags with some content between them:

<opening tag>content</closing tag>

These tags affect the way this content will be displayed. For example, the b-element makes the text look bold. The em-element makes the text look italic. The a-element creates the link to the other web pages or particular places on the web page, making the content clickable like this. The structure of this element is more complicated and includes the attribute (additional info) in the opening tag as follows:

<a href="url">

where url is the address of the link.

Some elements have no closing tags. For example, the br-element simply breaks the line so that the content after the single br-tag is displayed on the next line. The img-element displays the image that is hosted somewhere. This "somewhere" is described in the src-attribute as follows:

<img src="url">

The structure of the img-element can also include the alt-attribute:

<img src="url" alt="text">

This text will be displayed if there is no chance to display the image. In either case, there is no closing tag!

Some elements deal with the content, which is "bigger" than symbols, words, or lines. For example, the p-element defines a paragraph. Everything that is inside the p looks like a separated group. The same is about the div-element, which is a so-called box container. If I am not mistaken, the main idea behind containers is to add to the "macromanagement", whereas "micromanagement" is on inline elements :)

Web Page Philosophy

A web page has two dimensions: the one is the code, the other one is how a client sees it. From the "codding" point of view, there are three main things that matter: HTML, CSS, and DOM. With the help of the HTML, a developer adds the content to the page, sets the structure of this content, and connects the page to the CSS.

CSS - Cascading Style Sheets - are "responsible" for the style: fonts, colors, shapes, positioning, etc. It is possible to set the styling rules for various classes of elements within the CSS. Then, if the HTML-element is attributed to a particular class, it follows the styling rules defined for this class.

DOM - Document Object Model - is a standard convention for representing and interacting with the HTML-elements. Browsers percept HTML documents as tree-like structures. Besides other things, it means there is a hierarchy of elements. For example, if we have the em-element inside of the b-element, the text of the em-element is not only italic but also bold.

It's a good idea to percept a web page as the combination of boxes. This abstraction helps us understand the structure of the page better. One can imagine lots of different boxes. Three major ones are header box, main content box, and footer box. Moving from this general scheme forwards, one can also divide each of these areas into smaller boxes.

CSS Basics

The principal scheme of the CSS working includes selectors, properties, and declarations. For example:

h1 {
    background-color: #ffffff;
}

In this case, "h1" is the selector. The selector defines the element, to which the styling rule is applied. It may be the specific tag like here. It can also be the class of elements, or id, or the tag of the specific class, and so on. For example:

.image {
    background-color: #ffffff;
}

In this case, the rule is applied to any element of the "image" class. In both examples, "background-color" is the property. There are lots of different ones, like font-family, font-weight, margin, padding, border, position, display, and many many more. Every CSS property is a specific styling aspect. The name of the property usually gives a great hint of what it is all about. In these examples, CSS sets the color of the background.

Finally, the expression like "background-color: #ffffff" is the declaration. The declaration is the assignment of the value to the property. In these examples, the color of the background was defined as white. The same way one could set the styling rules regarding nearly everything in the HTML document using CSS.

It is a good idea to have a separate .css file for styling rule. This approach helps separate the style from the structure and simplifies the understanding of the code. To connect this CSS file to the HTML file, one has to include the link like this in the head section of the HTML document:

<link rel="stylesheet" href="name.css">

It is important that both HTML and CSS files are stored on the same level of the storage. Otherwise, the link must include the whole way to the CSS file to make it work.
